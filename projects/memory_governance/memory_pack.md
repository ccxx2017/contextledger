# 记忆包_记忆治理协议

## 概览
本项目为长对话记忆治理协议的设计与落地。当前已完成v1协议设计，进入试点执行阶段。首场冷启动已完成，标准循环工作流已定稿。

## 长期记忆
- 记忆包四节结构：概览、长期记忆、脊柱、开放问题。
- 锚点格式：`raw/raw_<起始日>_<主题>.md#turn<NNN>`（三位补零）。
- 每累计 **3 轮**对话触发一次折叠（全局唯一真源）。
- 技能卡必须包含 `source:` 字段，标明出自哪个项目。
- raw 文件名以主题线起始日命名，建立后永不改名，否则锚点断链。
- 全局技能库采用 **append-only** 模式，每次只追加新卡，不重写全文。
- 记忆包更新采用全文重写 + 行数核对（长期记忆和脊柱行数 ≥ 上一版，失效标记除外）。
- 维护提示词必须自包含，不引用模型看不到的文档（如 README）。
- 首场冷启动可跳过 3 轮节奏，直接处理全部历史桥稿。
- 日常对话 LLM 不能直接喂原始记忆包，需投影（去锚点、保留脉络文字）。
- 闭环内容四分流：通用可复用 → 技能卡；项目内长期有效 → 长期记忆；仍影响推进 → 稳定区；一次性细节 → 不保留（raw 已存）。
- 新旧冲突时标记 `【已失效】` 并注明取代项，不得静默删除。
- 折叠操作输入为 4 份：`maintenance_prompt.md`（指令）、本轮一次压缩桥稿（或 raw 原文）、当前记忆包全文、全局技能库全文。
- **首场冷启动流程**：将存量历史压缩稿存为 raw（标注“原始对话未留存”），备齐 4 份输入（新料 = 整份历史稿，memory_pack 为空，skills 为空），接收双产物后人工入库。
- **标准循环工作流**：每轮结束存 raw（`## turnNNN`）；满 3 轮触发折叠：取 3 轮 raw 原文做一次压缩生成桥稿（可跳过），喂 4 份输入；接收产物 A（记忆包全文）核对行数后整体替换，产物 B（技能卡草稿）审核后 append 进 skills；桥稿丢弃。
- **开新会话时的喂法**：`memory_pack.md` + 上次折叠后尚未折叠的 raw 段落（最多 2 轮）。
- **记忆包投影规则**（喂给日常对话 LLM）：去锚点投影 = 概览 + 长期记忆 + 开放问题原样 + 脊柱（仅脉络句，剥掉 `raw/...#turn`）。只读不回写。
- **人工核对行数方法**：只数「长期记忆」节和「脊柱」节的行数，新版本行数 ≥ 上一版（`[DEPRECATED]` 除外）。

## 开放问题
- 两周体检指标的具体数值（底稿长度趋稳标准、技能卡跨项目复用次数）待试点后定义。
- 何时将长期记忆升格为 wiki 结构？触发信号：某项目长期记忆超二三十行，或领域事实跨项目重复落，或概念查询需求出现。
- 全局技能库的“伪全局”风险待试点验证：是否有技能卡被来源项目之外实际用过？
- 首场冷启动生成的技能卡草稿需人工审核后入库，审核标准（通用可复用 vs 一次性）需在判定日志中显式记录。
- 投影规则的手动去锚点操作是否足够，还是需要日后脚本化？当前暂不脚本化。
