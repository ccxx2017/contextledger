## 二次压缩：生成上下文交接单
**触发场景**：长对话的一次压缩稿过长，不适合反复喂给大模型。
**操作步骤**：
1. 将一次压缩稿中的“过程叙述”转为“当前状态”。
2. 保留：当前目标、有效状态、关键约束、已完成修复、废弃方案、未解决问题、重要文件、下一步。
3. 删除过程细节、中间猜测、多次试错、大段代码。
4. 区分有效、废弃和待解决问题。
**已验证经验**：从“过程归档”转向“状态续作”可大幅减少 token，适合日常续接。
**适用边界**：适用于需要长期连续对话且历史越积越长的场景。
**来源**：memory_governance/projects/记忆治理协议/raw/raw_2026-06-12_compression.md#turn001

## 有界上下文结构
**触发场景**：需要压缩对话历史但希望总长度趋于常数。
**操作步骤**：
1. 划分【稳定区】：目标、关键约束、已闭环结论、脉络脊柱（每阶段一行）。
2. 划分【近况区】：只保留最近 N 轮详细压缩（N=3 可调）。
3. 每加一轮新内容，将最旧一轮折叠进稳定区：抽出脊柱一行 + 新结论。
**已验证经验**：稳定区增长为次线性（每轮只加一行），总长度有界。
**适用边界**：任何需要“保留脉络但不无限膨胀”的对话压缩。
**来源**：memory_governance/projects/记忆治理协议/raw/raw_2026-06-12_compression.md#turn002

## 闭环内容四分流规则
**触发场景**：一段对话已经闭环（问题已解决、方案已定），需要决定其去向。
**操作步骤**：
- 通用可复用 → 晶化为技能卡（入全局技能库）
- 项目内长期有效 → 写入项目记忆包的【长期记忆】节
- 仍影响当前推进 → 留在【稳定区】
- 一次性过程细节 → 不保留（原始证据已存 raw）
**已验证经验**：闭环不是删除资格，而是晶化资格。
**适用边界**：所有折叠操作中处理已闭环内容的场景。
**来源**：memory_governance/projects/记忆治理协议/raw/raw_2026-06-12_compression.md#turn005

## 逆流降级规则
**触发场景**：已写入长期记忆或技能库的内容被后续对话推翻。
**操作步骤**：
1. 原条目标记 `【已失效】`，不得静默删除。
2. 注明被哪条结论取代（附取代项来源锚点）。
3. 失效条目移入原稿证据层留痕，但仍保留在原位置（仅追加标记）。
**已验证经验**：技能库和记忆库必须支持“可逆”，否则会沉淀过期经验。
**适用边界**：所有长期记忆和技能卡的维护过程。
**来源**：memory_governance/projects/记忆治理协议/raw/raw_2026-06-12_compression.md#turn007

## 显式判定日志
**触发场景**：每次做分流/晶化决策后。
**操作步骤**：
1. 在技能库末尾“判定日志”节追加一行。
2. 格式：`YYYY-MM-DD：<决策内容>（什么进技能、什么进记忆、什么算一次性、触发晶化的原因）`
**已验证经验**：显式写出的判定规则比档案本身更重要，是将来交给自动化智能体的 SOP。
**适用边界**：手动维护记忆治理的任何阶段。
**来源**：memory_governance/projects/记忆治理协议/raw/raw_2026-06-12_compression.md#turn007

## 技能卡 append-only 模式
**触发场景**：更新全局技能库时。
**操作步骤**：
1. 模型只输出新增技能卡（产物B），不重写 skills 全文。
2. 人工审核后，将新卡追加到 `技能库.md` 末尾。
3. 老卡片碰都不碰，避免静默改写。
**已验证经验**：技能库是全局资产，全文重写风险极高，追加模式安全。
**适用边界**：任何多项目共享的技能库维护。
**来源**：memory_governance/projects/记忆治理协议/raw/raw_2026-06-12_compression.md#turn017

## 维护提示词自包含原则
**触发场景**：编写供 LLM 执行的维护提示词。
**操作步骤**：
1. 所有执行规则（数字、结构、锚点格式）必须内联写进 prompt。
2. 不引用模型看不到的文档（如 README、模板文件）。
3. 若需引用，只能“指路”而不能复制规则。
**已验证经验**：模型看不到项目文档，引用会导致指针断链。
**适用边界**：任何为 LLM 设计指令模板的场景。

## 首场冷启动流程
**触发场景**：现有存量历史压缩稿（如长对话的一次压缩汇总）需要首次生成记忆包和技能库基线。
**操作步骤**：
1. 将存量历史稿存为 raw 文件，文件顶部标注“本文件为一次压缩稿，原始对话未留存”。
2. 备齐 4 份输入：`maintenance_prompt.md`（指令）、新料 = 整份历史稿、`memory_pack.md` = 空壳（不存在则按四节从零生成）、`skills.md` = 空或仅表头。
3. 调用折叠 LLM，接收产物 A（记忆包全文）和产物 B（技能卡草稿）。
4. 人工入库：memory_pack 直接落盘（跳过行数核对），技能卡逐张审核后 append 进 skills，每张带 `source:` 字段。
5. 在 skills 末尾判定日志写一行记录。
**已验证经验**：首场冷启动不按 3 轮节奏回放历史，一次性建立基线。
**适用边界**：任何项目首次接入记忆治理协议时。
**来源**：memory_governance/projects/记忆治理协议/raw/raw_2026-06-13_workflow_confirmation.md#turn001

## 折叠操作标准循环
**触发场景**：日常对话积累满 3 轮，需要折叠入库。
**操作步骤**：
1. 每轮结束：将原始对话 append 进当前主题线的 raw 文件，用 `## turnNNN` 分节。
2. 满 3 轮时：取这 3 轮 raw 原文，做一次标准轮次压缩（保留逻辑链、去对话体、加标题），产出临时桥稿（不落盘）。若 3 轮很短可直接喂 raw 原文。
3. 备齐 4 份输入：`maintenance_prompt.md` + 桥稿（新料） + `memory_pack.md` 全文（存量） + `skills.md` 全文（参照）。
4. 调用折叠 LLM，接收产物 A（记忆包全文）和产物 B（技能卡草稿或“本轮无新增”）。
5. 人工核对：数产物 A 的「长期记忆」节和「脊柱」节行数，新 ≥ 旧（`[DEPRECATED]` 除外），通过则整体替换 memory_pack。
6. 技能卡草稿审核后 append 进 skills（带 source），判定日志写一行。
7. 丢弃桥稿。
**已验证经验**：无近况区，记忆包只存状态和脊柱，细节全在 raw；开新会话时喂 memory_pack + 未折叠的尾部 raw（≤2 轮）。
**适用边界**：所有采用本协议的项目日常维护。
**来源**：memory_governance/projects/记忆治理协议/raw/raw_2026-06-13_workflow_confirmation.md#turn001

## 记忆包投影规则
**触发场景**：将 memory_pack 喂给日常对话 LLM（非折叠 LLM）。
**操作步骤**：
1. 从 memory_pack 中提取：概览节、长期记忆节、开放问题节（均原样保留）。
2. 从脊柱节提取脉络句（每行中描述做什么、为何转向、结果的部分），删除行尾的锚点（`〔raw/...#turnNNN〕` 及 `→ skills: ...`）。
3. 将上述四部分拼接成临时上下文，喂给日常对话 LLM。
4. 该投影为只读、不落盘、不回写。
**已验证经验**：锚点是给人和检索系统的，日常对话 LLM 无法访问 raw 文件，锚点在其上下文中是死字符串，必须去掉。
**适用边界**：任何需要将记忆包用于一般对话续接的场景。
**来源**：memory_governance/projects/记忆治理协议/raw/raw_2026-06-13_workflow_confirmation.md#turn001
